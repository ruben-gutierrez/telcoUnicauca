Arquitectura Clearwater IMS

Núcleo IMS de código libre desarrollado por Metaswitch Networks bajo licencia GNU GPLv3. Clearwater es un proyecto desarrollado para ser escalable horizontal y modularmente en un entorno virtualizado garantizando balanceo de carga entre sus componentes. En la figura 1 se visualiza la arquitectura Clearwater IMS desplegado en un sistema distribuido.

Un sistema distribuido permite instanciar cada nodo de Clearwater (IBCF, bono, ellis, dime, Homer, Vellum y Sprout) en máquinas virtuales (MV) individuales orquestadas por la plataforma Telco IMS proporcionando una infraestructura de red más completa. No obstante, Clearwater también puede ser implementado en un sistema todo en uno instanciado en una sola MV para pruebas de concepto. Las funciones principales de cada nodo son explicadas a continuación:

  • Bono (Proxy de Borde): primer punto de contacto de los equipos de usuario con el núcleo IMS de Clearwater que incluye funciones de proxy de borde P-CSCF escalable horizontalmente. Este nodo, es el encargado de procesar solicitudes SIP de los clientes IMS y brindar soporte NAT.
  • Sprout (Router SIP): Registrador y servidor proxy que provee servicios de autenticación de clientes y enrutamiento de mensajes SIP a los servidores de aplicación por medio de funciones de red I-CSCF y S-CSCF incorporadas internamente. El nodo Sprout es escalable horizontalmente y es el encargado de realizar el balanceo de carga entre los nodos bono a partir de un servidor DNS.
  • Dime: brinda soporte de interconexión con la base de datos HSS mediante señalización diameter. Dime proporciona una interfaz de servicios web para que Sprout pueda recuperar credenciales de autenticación y perfiles de usuario

  • Vellum (State store): Almacena estados de larga duración del núcleo IMS en clústeres de almacenamiento distribuidos y optimizados para servicios en la nube. Asimismo, cuando no se utiliza HSS, vellum almacena credenciales de autenticación e información de perfiles de usuario en cassandra, además de almacenar configuraciones de servicios.
  • Homer (XDMS): almacena archivos de configuración de los servicios para cada usuario en el sistema. Utiliza una interfaz XCAP estándar para crear, leer, actualizar o eliminar los archivos de configuración, además, de usar el nodo Vellum para almacenar datos de larga duración.
  • Ellis: proporciona gestión de administración de usuarios, contraseñas y aprovisionamiento de servicio a partir de una interfaz web. En el nodo ellis son creados los números de teléfono de los usuarios que van a utilizar los servicios de la arquitectura IMS. Este nodo no es escalable.
