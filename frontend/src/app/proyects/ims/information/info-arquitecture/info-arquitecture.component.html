Arquitecturas Ambiente de Pruebas Telco IMS

El ambiente de pruebas (Testbed) Telco IMS utiliza el sistema de código libre de Openstack para la gestión y orquestación de recursos virtuales en la red. Específicamente, Openstack proporciona soluciones de Infraestructura como servicios (IaaS) para que los usuarios puedan aprovisionar recursos de almacenamiento, procesamiento y redes virtuales para la prestación de servicios en la nube. Las arquitecturas proporcionadas en el testbed Telco IMS están compuestas por una red privada virtual donde se conectan las instancias de máquinas virtuales como se observa en la Figura 1. En este caso, las instancias son conectadas a la red 192.168.233.0/24 llamada “Shared” la cual está conectada a la red pública de Openstack por medio de un enrutador virtual que permite acceder a redes externas y servicios de internet

Nota: La plataforma Telco provee un acceso automatizado a las arquitecturas de MV creadas en la plataforma. Las MV conectadas en la red privada virtual utilizan el direccionamiento de esa red para configuración interna. No obstante, para acceder a las MV desde una red externa se hace por medio de una IP pública. Ver información de Openstack.

Información Máquinas virtuales

Para configurar y acceder a las MV es importante conocer su direccionamiento IP. En la Figura 2 y Figura 3 se observa los datos generales de una máquina virtual creada en la plataforma Telco IMS. El rango IP pública para acceso desde la red externa es de 192.168.40.x. Mientras que el rango IP de la red privada virtual depende de la arquitectura reservada con mascara de red /24. Para ver la dirección IP de la máquina virtual en la consola se puede hacer por medio del comando ifconfig.

Nota: La dirección IP pública es utilizada para acceder a las dashboard de configuración de Clearwater IMS y configurar los equipos softphone.

Arquitectura Clearwater IMS Distribuido

Núcleo IMS de código libre desarrollado por Metaswitch Networks bajo licencia GNU GPLv3. Clearwater es un proyecto desarrollado para ser escalable horizontal y modularmente en un entorno virtualizado garantizando balanceo de carga entre sus componentes. En la figura 1 se visualiza la arquitectura Clearwater IMS desplegado en un sistema distribuido.

Un sistema distribuido permite instanciar cada nodo de Clearwater (IBCF, bono, ellis, dime, Homer, Vellum y Sprout) en máquinas virtuales (MV) individuales orquestadas por la plataforma Telco IMS. La arquitectura Clearwater IMS distribuida utiliza un servidor DNS para identificación de los nodos mediante dominios URL. El servidor DNS permite disminuir la carga de trabajo en los servidores y los retardos producidos por las consultas. Asimismo, permite aumentar la tolerancia a fallos en el sistema. Arquitectura Clearwater IMS todo en uno (AIO)

Si bien Clearwater es un proyecto desarrollado para ser escalable horizontal y modularmente de forma masiva, Clearwater también proporciona un sistema todo en uno (AIO) que permite instalar todos sus componentes de red en una sola máquina virtual. Este sistema provee una plataforma que permite familiarizarse con la arquitectura Clearwater IMS antes de pasar a una escala mayor. Asimismo, permite realizar pruebas de concepto sobre su arquitectura y utilizar servicios básicos de llamada y videollamada. Para ello, es necesario configurar los usuarios por medio de la Dashboard de Clearwater en la web la cual se accede mediante la IP pública de la máquina virtual.

Clearwater AIO es instalado sobre una MV con sistema operativo Ubuntu 14 configurado para usar direccionamiento dinámico DHCP, además de utilizar los servicios de red de Bono, Sprout, Homestead, Homer y Ellis. Clearwater debe puede ser ejecutado en una MV con un solo núcleo, 2GB de RAM y 10 GB de almacenamiento.

Restricciones

  • * Utilización de dominio SIP URI predeterminado por defecto "example.com".
  • * Menor rendimiento que un sistema a escala mayor debido a que todos sus componentes son instalados en una sola MV.
  • * Este sistema no se puede ser escalado agregando más MV en la red para aumentar su capacidad. Para ello, es necesario instalar el sistema Clearwater Distribuido.
  • * No proporciona tolerancia a fallos, dado que se ejecuta todo el sistema en una sola MV que si falla genera que todo el servicio en conjunto falle.
